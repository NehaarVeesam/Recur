Statement:
You are given an integer n. Your task is to compute the GCD (greatest common divisor) of two values:

sumOdd: the sum of the smallest n positive odd numbers.

sumEven: the sum of the smallest n positive even numbers.

Return the GCD of sumOdd and sumEven.

Approach: Calculating GCD of 2 numbers
Time Complexity: O(logn)
Space Complexity: O(1)
1. Calculate the odd and even sum 
2. Find out GCD

GCD of 2 numbers 
Example 
16, 20 
remainder of (20,16) -> 4 , then it becomes (16,4) -> remainder is 0 

So simply when the divisor becomes 0 u exit 
and in loop we calculate se, so = so , se % so
at last answer is stored at se

Time complexity why logn -> 
The Euclidean algorithm runs in O(logn) because the numbers shrink very quickly after each modulo operation.

Approach: Math solve
Time Complexity: O(1)
Space Complexity: O(1)
Sum of odd numbers -> n*n 
Sum of even numbers -> n*(n+1)

As n is already common , cancel out 
gcd (n , n+1) is 1
Therefore answer becomes n

Learning:
1. Sum of first n odd numbers = n²
2. Sum of first n even numbers = n(n+1)
3. gcd(a,b)=gcd(b,amodb)
4. Recognize hidden O(1) solutions

Your first instinct was to implement the Euclidean algorithm, which is correct. But sometimes the problem is designed so that mathematical reasoning reduces the entire computation to a constant-time answer.

# GCD Approach 
class Solution:
    def gcdOfOddEvenSums(self, n: int) -> int:
        so = n * n
        se = n * (n + 1)

        while so:
            se, so = so, se % so
        return se
